# 高斯泼溅建模链路补强方案 v2.0

## 1. 方案目标

构建一套面向视频输入、无人机采集和大场景重建的自动化高斯泼溅建模链路。系统通过浏览器接入视频数据，完成视频检查、样本增强、自动抽帧、相机位姿求解、三维高斯训练、模型导出、增量渲染、浏览器加载和质量诊断，形成可交互查看、可持续优化、可追溯复盘的实景三维模型。

链路目标包括：

```text
视频可用性判断
自动抽帧与关键帧选择
低采集样本 AI 增强
高成功率相机注册
建模效率加速
稳定三维高斯训练
模型分级导出
增量渲染与渐进式加载
浏览器实时查看
失败原因诊断
重建质量评估
无人机场景适配
大场景分块与流式展示
全链路命令与产物追溯
```

---

## 2. 总体技术链路

```text
视频上传
→ 视频元数据解析
→ 视频质量预检
→ 可重建性评分
→ 自适应抽帧
→ 坏帧清洗
→ 关键帧选择
→ 低采集样本 AI 增强
→ 相机位姿求解
→ 位姿质量门禁
→ 建模效率加速调度
→ 3DGS 训练
→ 阶段性模型导出
→ 增量渲染与渐进式加载
→ 模型质量检查
→ Gaussian Splat 导出
→ 浏览器格式转换
→ SparkJS / Three.js 加载
→ 三维场景交互查看
→ 诊断结果与建模报告沉淀
```

系统整体采用“先可见、后精细；先局部、后全局；先粗模、后高质”的处理方式，使建模过程不再是一次性等待完整结果，而是形成连续可反馈的自动化重建链路。

## 3. 视频接入层

### 3.1 视频上传

浏览器端提供视频上传入口，支持本地视频文件上传。

支持格式：

```text
MP4
MOV
AVI
MKV
```

上传后生成视频资产记录：

```text
video_id
filename
file_size
duration
resolution
fps
codec
bitrate
upload_time
storage_path
sha256
```

### 3.2 视频元数据解析

后端通过 `ffprobe` 解析视频基础信息。

解析字段包括：

```text
视频时长
视频分辨率
帧率
编码格式
码率
音视频流信息
旋转角度
色彩空间
关键帧分布
```

解析结果写入任务元数据，并作为后续质量预检、抽帧策略、低样本判断、训练预算和导出策略的依据。

## 4. 视频质量预检层

### 4.1 基础质量检查

对上传视频执行自动预检。

检查项包括：

```text
视频时长是否满足建模要求
分辨率是否达到最低要求
帧率是否稳定
视频是否损坏
编码格式是否可解码
画面是否存在严重过曝或欠曝
画面是否存在大面积纯色区域
画面是否存在剧烈抖动
画面是否存在快速旋转
画面是否存在大面积动态物体
```

### 4.2 可重建性评分

系统对视频生成可重建性评分。

评分维度包括：

```text
画面清晰度
帧间重叠度
视角变化连续性
场景纹理丰富度
动态物体占比
曝光稳定性
运动模糊比例
可用关键帧比例
预计相机注册成功率
```

输出结果：

```text
reconstructability_score
blur_score
overlap_score
motion_score
texture_score
dynamic_ratio
exposure_score
usable_frame_ratio
```

### 4.3 预检结果分级

预检结果分为：

```text
PASS：进入抽帧和建模流程
WARN：允许继续，但记录风险项
FAIL：阻断建模，返回失败原因
```

失败原因结构化保存：

```text
error_code
error_stage
error_reason
evidence_frame
diagnostic_detail
```

## 5. 自适应抽帧层

### 5.1 抽帧目标计算

系统根据视频时长、帧率、预设模式、画面运动量和质量评分动态计算抽帧目标。

抽帧参数包括：

```text
target_frame_count
min_frame_interval
max_frame_interval
sampling_fps
motion_threshold
blur_threshold
similarity_threshold
```

不同任务模式对应不同抽帧目标：

```text
fast：低帧数，快速预览
standard：中等帧数，稳定建模
high：高质量关键帧，质量优先
diagnostic：用于失败诊断和补强注册
```

### 5.2 时间均匀采样

第一轮抽帧采用时间均匀采样，保证视频全程覆盖。

输出：

```text
raw_frames/
frames_manifest.json
```

`frames_manifest.json` 记录：

```text
frame_id
timestamp
source_frame_index
image_path
width
height
blur_score
motion_score
similarity_score
selected_status
drop_reason
```

### 5.3 运动自适应采样

系统根据画面运动量调整抽帧密度：

```text
运动慢的片段降低抽帧密度
运动快但画面稳定的片段提高抽帧密度
剧烈抖动片段降低权重
快速旋转片段降低权重
高重叠稳定片段保留关键帧
```

## 6. 坏帧清洗层

### 6.1 模糊帧过滤

通过图像清晰度算法识别模糊帧。

过滤类型包括：

```text
运动模糊
对焦失败
压缩噪声严重
画面拖影
低光噪点严重
```

### 6.2 重复帧过滤

通过相似度计算移除高度重复帧。

处理逻辑：

```text
计算相邻帧视觉相似度
保留视角变化有效的帧
移除几乎无变化的连续帧
避免大量重复帧进入 COLMAP
```

### 6.3 无效画面过滤

移除不适合参与建模的帧：

```text
大面积天空
大面积地面
大面积水面
大面积白墙
严重过曝
严重欠曝
强反光画面
遮挡画面
纯动态人群画面
快速转场画面
```

### 6.4 动态区域标记

识别视频中的动态区域，并生成动态掩码。

动态区域包括：

```text
行人
车辆
树叶
水面
烟雾
旗帜
强阴影变化
临时移动物体
```

输出：

```text
dynamic_masks/
dynamic_ratio.json
```

动态掩码用于后续相机注册、训练降权和质量诊断。

## 7. 关键帧选择层

### 7.1 关键帧评分

每一帧生成综合评分。

评分字段：

```text
sharpness_score
overlap_score
viewpoint_score
texture_score
exposure_score
dynamic_penalty
coverage_score
final_score
```

### 7.2 关键帧筛选

系统按照以下规则筛选关键帧：

```text
保证时间覆盖
保证空间视角变化
保证相邻帧重叠
保证清晰度
减少重复画面
减少动态干扰
保留闭环视角
```

### 7.3 关键帧分组

关键帧按时间和视角关系分组：

```text
primary_sequence：主连续序列

side_view_candidates：侧向视角候选帧
weak_texture_segments：弱纹理片段
risk_segments：高风险片段
```

## 8. 建模效率加速层

### 8.1 快速预览优先

系统采用“快速预览模型先出，高质量模型后台继续优化”的两阶段建模策略。

```text
视频上传
→ 快速抽帧
→ 快速相机注册
→ 低迭代 Splatfacto 训练
→ 快速导出预览模型
→ 浏览器先加载预览版
→ 后台继续高质量训练
→ 高质量模型完成后自动替换
```

模型产物分为：

```text
preview_model：快速预览模型
standard_model：标准质量模型
high_model：高质量模型
```

### 8.2 关键帧优先训练

训练阶段不直接使用所有抽帧结果，而是优先选择高质量关键帧。

关键帧筛选依据：

```text
清晰度
纹理丰富度
视角覆盖
帧间重叠
动态区域占比
曝光稳定性
闭环价值
```

该策略减少无效帧进入 COLMAP 和训练流程，提高相机注册速度、训练速度和模型收敛效率。

### 8.3 多阶段训练

训练分为三个阶段：

```text
coarse stage：快速建立粗模型
refine stage：优化核心区域细节
polish stage：提升边缘、纹理和颜色稳定性
```

每个阶段可以独立输出中间模型，供前端提前查看。

### 8.4 训练提前停止

系统持续监控训练指标：

```text
loss 收敛速度
PSNR / SSIM 变化
高斯数量增长速度
模型质量提升幅度
训练耗时
显存占用
```

当模型质量提升低于阈值时，提前停止训练，避免无效计算。

### 8.5 任务缓存与复用

对同一视频或同一批帧重复建模时，复用已有中间产物：

```text
视频元数据
抽帧结果
清洗结果
COLMAP database
相机位姿
transforms.json
动态 mask
训练配置
```

### 8.6 GPU 单任务调度

本地 RTX GPU 采用单任务重建锁，避免多个重建任务同时争抢显存。

调度状态包括：

```text
gpu_idle
gpu_reserved
gpu_running
gpu_releasing
gpu_failed
```

排队任务只进入等待队列，不同时启动训练。

### 8.7 分辨率递进训练

系统按分辨率递进训练：

```text
低分辨率快速建立结构
中分辨率补强整体细节
高分辨率优化核心区域纹理
```

低分辨率阶段优先产出 preview，高分辨率阶段产出 standard 或 high。

## 9. 低采集样本 AI 增强层

### 9.1 低样本识别

系统在抽帧、清洗和注册前判断当前视频是否属于低采集样本。

判断指标：

```text
可用关键帧数量不足
有效视角覆盖不足
闭环视角不足
侧向视角不足
纹理区域不足
相机轨迹过短
COLMAP 注册率偏低
```

输出：

```text
low_sample_status
low_sample_reason
missing_view_type
coverage_gap
pose_risk_score
```

### 9.2 学习式位姿补强

当 COLMAP 注册不足时，引入 AI 位姿和匹配增强模块。

可接入模块：

```text
DUSt3R
MASt3R
VGGSfM
LoFTR
RoMa
SuperGlue
```

处理流程：

```text
低样本视频
→ 提取关键帧
→ 学习式特征匹配
→ 生成候选相机关系
→ 补强 COLMAP 匹配
→ 输出更完整相机位姿
→ 进入 3DGS 训练
```

### 9.3 深度先验增强

在低样本情况下，通过单目深度或多视角深度估计生成几何先验。

可生成：

```text
depth map
normal map
scene boundary
surface prior
scale prior
```

这些先验用于：

```text
辅助高斯初始化
约束场景几何
减少漂浮点
降低空洞
提升弱纹理区域稳定性
```

### 9.4 视角补全增强

对于视角覆盖不足的区域，使用 AI 生成候选补全信息。

处理对象：

```text
缺少侧面视角的建筑立面
缺少背面视角的物体
弱纹理墙面
重复纹理区域
边缘遮挡区域
```

补全结果作为训练辅助先验：

```text
辅助几何连续性
辅助颜色一致性
辅助空洞区域补齐
辅助 viewer 预览完整性
```

### 9.5 超分与去模糊增强

低质量帧进入训练前可进行图像增强。

增强类型：

```text
去模糊
降噪
超分辨率
曝光修正
白平衡修正
压缩伪影修复
```

增强后的帧保留原始帧映射关系：

```text
original_frame
enhanced_frame
enhancement_type
enhancement_model
confidence_score
```

### 9.6 不确定性标记

AI 增强区域必须标记不确定性。

标记类型：

```text
ai_enhanced_area
low_confidence_geometry
inferred_texture
missing_view_filled
synthetic_prior_used
```

viewer 中可显示：

```text
真实采集区域
AI 增强区域
低置信区域
缺视角补全区域
```

### 9.7 增强产物追溯

AI 增强相关产物独立保存。

产物包括：

```text
enhanced_frames/
depth_priors/
normal_priors/
pose_candidates/
ai_masks/
uncertainty_map.json
enhancement_report.json
```

每个增强产物记录：

```text
source_frame
enhancement_method
model_name
model_version
confidence
created_at
used_in_training
```

## 10. 相机位姿求解层

### 10.1 COLMAP 视频模式

视频输入使用顺序匹配策略。

核心配置：

```text
single_camera = true
camera_model = SIMPLE_RADIAL / OPENCV
matcher = sequential_matcher
guided_matching = true
loop_detection = true
```

流程：

```text
feature_extractor
→ sequential_matcher
→ mapper
→ image_undistorter
→ transforms.json 生成
```

### 10.2 COLMAP 参数增强

针对视频建模增强以下参数：

```text
提高 SIFT 特征数量
开启 guided matching
增加 sequential overlap
开启 loop detection
控制最大图像尺寸
统一相机内参
启用畸变模型
保留匹配日志
```

### 10.3 注册结果解析

解析 COLMAP 和 Nerfstudio 输出结果。

关键指标：

```text
input_frame_count
registered_frame_count
registration_ratio
camera_count
sparse_point_count
mean_reprojection_error
track_length_mean
coverage_ratio
failed_frame_count
```

### 10.4 位姿质量门禁

相机位姿求解后执行门禁检查。

检查项：

```text
注册帧数量
注册帧比例
重投影误差
相机轨迹连续性
稀疏点数量
场景覆盖范围
是否存在轨迹断裂
是否存在大量孤立帧
```

门禁结果：

```text
PASS：进入训练
WARN：进入训练并记录风险
FAIL：停止训练并输出诊断
```

## 11. 位姿求解增强层

### 11.1 自动重采样

当注册结果不达标时，系统自动触发重采样流程。

重采样策略：

```text
降低抽帧数量
提高关键帧质量阈值
增加帧间间隔
强化闭环候选帧
移除弱纹理片段
移除高动态片段
调整 sequential overlap
```

### 11.2 多策略注册

系统保留多套 COLMAP 注册策略。

策略类型：

```text
video_sequential
video_sequential_loop
video_exhaustive_small
video_vocab_tree
high_quality_keyframes
low_motion_keyframes
```

每次注册产出独立目录：

```text
colmap_attempt_001/
colmap_attempt_002/
colmap_attempt_003/
```

系统根据注册质量选择最优结果进入训练。

### 11.3 学习式位姿 fallback

当 COLMAP 注册失败或覆盖不足时，进入学习式位姿 fallback 流程。

可接入模块：

```text
DUSt3R
MASt3R
VGGSfM
RoMa
LoFTR
SuperGlue
```

补强流程：

```text
COLMAP 初次注册
→ 注册质量检查
→ 学习式匹配 / 位姿估计
→ 生成候补相机位姿
→ 对齐相机坐标系
→ 质量检查
→ 输出 transforms.json
```

## 13. 增量渲染与渐进式加载层

### 13.1 增量可见机制

系统不等待完整模型全部生成后才展示，而是按阶段逐步向 viewer 输出可加载模型。

输出顺序：

```text
粗略点云预览
→ 低精度 Gaussian preview
→ 核心区域高斯模型
→ 周边区域高斯模型
→ 完整标准模型
→ 高质量模型
```

前端 viewer 支持模型热更新：

```text
加载初始模型
接收新 tile
替换旧 tile
更新高斯数量
更新模型质量状态
保持当前相机视角
```

### 13.2 分块增量导出

大场景按 tile 输出，不等待整场景全部完成。

每个 tile 独立记录：

```text
tile_id
tile_status
tile_bounds
splat_count
model_url
lod_level
quality_score
updated_at
```

tile 状态：

```text
pending
training
exported
loaded
updated
failed
```

### 13.3 前端按需加载

viewer 根据用户视角和距离加载模型：

```text
当前视野内 tile 优先加载
相机附近 tile 优先加载
核心区域 tile 优先加载
远处区域加载低精度版本
视野外 tile 延迟加载
```

加载策略：

```text
near_field：高精度
mid_field：中精度
far_field：低精度
out_of_view：不加载或卸载
```

### 13.4 渐进式质量替换

同一区域可以存在多个质量版本：

```text
tile_001_preview
tile_001_standard
tile_001_high
```

替换逻辑：

```text
先加载 preview
standard 完成后替换 preview
high 完成后替换 standard
替换时保持坐标系不变
替换时不重置用户视角
```

### 13.5 增量训练联动

训练端支持阶段性导出中间结果。

流程：

```text
训练开始
→ 达到 preview 迭代数
→ 导出 preview splat
→ 前端加载
→ 继续训练
→ 达到 standard 迭代数
→ 导出 standard splat
→ 前端替换
→ 继续高质量优化
→ 导出 high splat
→ 前端最终替换
```

### 13.6 渲染状态反馈

前端展示当前模型状态：

```text
当前加载质量等级
已加载 tile 数量
总 tile 数量
当前高斯数量
模型更新时间
后台是否仍在优化
当前是否为最终模型
```

显示示例：

```text
当前模型：Preview
后台状态：High 模型训练中
已加载：8 / 24 tiles
Splats：1,968,696
下一次更新：核心区域 high tile 完成后自动替换
```
