我按你们这个“多路无人机 + 预建底座 + 实时增量建模”场景筛了一批。结论是：第一批不要追 4DGS 论文，先跑通 **OpenDroneMap/COLMAP/Nerfstudio/gsplat/SuperSplat + 位姿/遥测对齐**。

**一、优先关注的开源项目**

| 方向            | 项目                                                         | 用途                                     | 判断                                      |
| --------------- | ------------------------------------------------------------ | ---------------------------------------- | ----------------------------------------- |
| 无人机基础建模  | [OpenDroneMap / ODM](https://github.com/OpenDroneMap/ODM)    | 无人机影像生成正射、点云、DEM、3D 模型   | 适合基础底图、工程化程度高                |
| SfM/MVS         | [COLMAP](https://github.com/colmap/colmap)                   | 从视频抽帧/照片求相机位姿、稀疏点云、MVS | 3DGS 前处理核心                           |
| SfM             | [OpenSfM](https://github.com/mapillary/OpenSfM)              | 支持 GPS/传感器辅助的 SfM                | 适合研究无人机遥测融合                    |
| 摄影测量        | [AliceVision / Meshroom](https://github.com/alicevision/AliceVision) | 开源摄影测量流水线                       | 可做对照方案                              |
| 摄影测量        | [MicMac](https://github.com/micmacIGN/micmac/wiki)           | 法国 IGN 开源摄影测量工具                | 偏测绘/专业，学习成本高                   |
| 3DGS 基线       | [graphdeco-inria/gaussian-splatting](https://github.com/graphdeco-inria) | 原始 3DGS 实现                           | 论文 baseline，不建议直接当工程主线       |
| 3DGS 工程化     | [Nerfstudio / Splatfacto](https://docs.nerf.studio/nerfology/methods/splat.html) | 训练 Gaussian splats、导出 `.ply`        | Demo 首选                                 |
| 训练颜色校正加速 | [fused-bilagrid](https://github.com/harry7557558/fused-bilagrid) | 用 fused CUDA kernel 替换 Nerfstudio bilateral grid 的 `slice` 和 `total_variation_loss` | 训练阶段性能优化储备；仅在启用 bilateral grid/曝光颜色校正时有价值 |
| 3DGS CUDA 库    | [gsplat](https://github.com/nerfstudio-project/gsplat)       | PyTorch + CUDA Gaussian rasterization    | 后续自研增量优化核心库                    |
| NVIDIA 工业链路 | [3DGRUT / 3DGUT](https://github.com/nv-tlabs/3dgrut)         | 支持畸变相机、鱼眼、滚动快门等复杂相机   | 对无人机相机模型很有价值                  |
| 浏览器编辑      | [SuperSplat](https://github.com/playcanvas/supersplat)       | 检查、裁剪、优化、发布 splat             | Demo 展示和清理首选                       |
| Web 渲染        | [SparkJS](https://github.com/sparkjsdev/spark)               | Three.js 里加载和渲染 3DGS               | 适合做三维态势前端                        |
| 3DGS 转换       | [splat-transform](https://github.com/playcanvas/splat-transform) | 3DGS 格式转换、压缩、处理                | 发布链路有用                              |
| Cesium/GIS      | [Cesium 3D Gaussian Splats / 3D Tiles](https://cesium.com/blog/2026/04/27/3d-gaussian-splats-lod/) | 3DGS 进入 3D Tiles/GIS 生态              | 和指挥系统最贴近，重点关注                |
| 单图 3DGS       | [Apple SHARP](https://github.com/apple/ml-sharp)             | 单张图快速生成 3DGS                      | 只适合研究/Demo，模型权重许可限制商业用途 |

补充判断：

- `fused-bilagrid` 不是完整 3DGS/4DGS 建模系统，也不负责抽帧、COLMAP 位姿、导出或浏览器展示；它只加速 Nerfstudio/Splatfacto 训练中的 bilateral grid 颜色/曝光校正模块。
- 适用场景是车载/无人机采集素材存在曝光变化、白平衡漂移或多视角颜色不一致，并且训练配置确实启用了 bilateral grid。
- 接入方式更适合放进 WSL2/Docker/Nerfstudio 训练镜像中验证，先用同一批素材对比 `train_splatfacto` 耗时、视觉一致性和导出模型质量；不作为当前 P0 “上传 -> 处理 -> 建模 -> 展示”任务页面的阻塞项。

**三、工业化/工程化论文优先读**

第一批建议按这个顺序读：

1. [3D Gaussian Splatting for Real-Time Radiance Field Rendering](https://arxiv.org/abs/2308.04079)  
   3DGS 原始论文，理解 `.ply`、相机位姿、Gaussian 优化的基础。

2. [gsplat: An Open-Source Library for Gaussian Splatting](https://arxiv.org/abs/2409.06765)  
   工程化 CUDA/PyTorch 库，后续自研增量建模绕不开。

3. [3DGUT: Enabling Distorted Cameras and Secondary Rays in Gaussian Splatting](https://arxiv.org/abs/2412.12507)  
   NVIDIA 方向，重点看复杂相机模型、鱼眼、滚动快门。无人机相机很容易踩这个坑。

4. [ORB-SLAM3](https://arxiv.org/abs/2007.11898)  
   实时视觉/视觉惯性 SLAM 基准。对无人机位姿补偿有参考价值。

5. [Kimera: Real-Time Metric-Semantic Localization and Mapping](https://arxiv.org/abs/1910.02490)  
   语义 + SLAM + 实时地图，很贴近“态势感知”。

6. [RTG-SLAM](https://arxiv.org/abs/2404.19706)  
   Gaussian Splatting 实时重建系统，适合理解“在线优化”怎么做。

7. [4D Gaussian Splatting for Real-Time Dynamic Scene Rendering](https://arxiv.org/abs/2310.08528)  
   动态复盘方向，但不要把它当一期可交付能力。

8. [Street Gaussians](https://arxiv.org/abs/2401.01339)  
   动态城市场景、自动驾驶数据，适合你们的人车动态复盘参考。

9. [HUGS: Holistic Urban 3D Scene Understanding via Gaussian Splatting](https://arxiv.org/abs/2403.12722)  
   同时做几何、外观、语义、运动，适合看系统设计。

10. [SHARP](https://arxiv.org/abs/2512.10685)  
      单图快速 3DGS，可作为现场关键帧快速空间化的支线能力。

**四、我建议你们先跑的组合**

POC 第一批就跑这套：

```text
OpenDroneMap / COLMAP
  -> 基础相机位姿和点云

Nerfstudio Splatfacto
  -> 从无人机视频抽帧生成 3DGS

SuperSplat
  -> 检查、裁剪、发布 splat

SparkJS / CesiumJS
  -> 浏览器三维态势展示

ORB-SLAM3 / OpenVINS / VINS-Fusion
  -> 验证无人机视频 + IMU/GPS 位姿融合
```

第二批再研究：

```text
gsplat
  -> 自研局部增量优化

3DGUT
  -> 处理鱼眼/畸变/滚动快门等无人机相机问题

MonoGS / RTG-SLAM / SplaTAM
  -> 研究实时 Gaussian SLAM

Street Gaussians / HUGS / EmerNeRF
  -> 研究静态背景 + 动态人车拆分
```

**五、关键判断**

当前最有工程价值的不是“实时 4DGS”，而是：

```text
预建基础三维底座
+ 多无人机实时位姿
+ 视频 AI 目标/事件
+ 局部变化检测
+ 局部增量 splat / mesh / point cloud
+ 原始视频证据回溯
```

所以开源项目优先级应该是：**测绘建图 > 位姿同步 > 3DGS 训练展示 > 局部增量 > 动态 4DGS**。
