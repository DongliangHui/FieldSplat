# 高斯泼溅建模链路补强方案

## 1. 方案目标

构建一套面向视频输入的自动化高斯泼溅建模链路，使用户通过浏览器上传视频后，系统能够完成视频检查、帧处理、相机位姿求解、三维高斯训练、模型导出、浏览器渲染和质量诊断，形成可交互查看的实景三维模型。

链路目标包括：

```text
视频可用性判断
自动抽帧与关键帧选择
高成功率相机注册
稳定三维高斯训练
模型压缩与浏览器加载
失败原因诊断
重建质量评估
无人机场景适配
大场景分块与流式展示
```

------

## 2. 总体技术链路

```text
视频上传
→ 视频元数据解析
→ 视频质量预检
→ 自适应抽帧
→ 坏帧清洗
→ 关键帧选择
→ 相机位姿求解
→ 位姿质量门禁
→ 3DGS 训练
→ 模型质量检查
→ Gaussian Splat 导出
→ 浏览器格式转换
→ SparkJS / Three.js 加载
→ 三维场景交互查看
→ 诊断结果与建模报告沉淀
```

------

## 3. 视频接入层

### 3.1 视频上传

浏览器端提供视频上传入口，支持本地视频文件上传。

支持格式：

```text
MP4
MOV
AVI
MKV
```

上传后生成视频资产记录：

```text
video_id
filename
file_size
duration
resolution
fps
codec
bitrate
upload_time
storage_path
sha256
```

### 3.2 视频元数据解析

后端通过 `ffprobe` 解析视频基础信息。

解析字段包括：

```text
视频时长
视频分辨率
帧率
编码格式
码率
音视频流信息
旋转角度
色彩空间
关键帧分布
```

解析结果写入任务元数据，并作为后续质量预检、抽帧策略和训练参数选择依据。

## 4. 视频质量预检层

### 4.1 基础质量检查

对上传视频执行自动预检。

检查项包括：

```text
视频时长是否满足建模要求
分辨率是否达到最低要求
帧率是否稳定
视频是否损坏
编码格式是否可解码
画面是否存在严重过曝或欠曝
画面是否存在大面积纯色区域
画面是否存在剧烈抖动
画面是否存在快速旋转
画面是否存在大面积动态物体
```

### 4.2 可重建性评分

系统对视频生成可重建性评分。

评分维度包括：

```text
画面清晰度
帧间重叠度
视角变化连续性
场景纹理丰富度
动态物体占比
曝光稳定性
运动模糊比例
可用关键帧比例
预计相机注册成功率
```

输出结果：

```text
reconstructability_score
blur_score
overlap_score
motion_score
texture_score
dynamic_ratio
exposure_score
usable_frame_ratio
```

### 4.3 预检结果分级

预检结果分为：

```text
PASS：进入抽帧和建模流程
WARN：允许继续，但记录风险项
FAIL：阻断建模，返回失败原因
```

失败原因结构化保存：

```text
error_code
error_stage
error_reason
evidence_frame
diagnostic_detail
```

## 5. 自适应抽帧层

### 5.1 抽帧目标计算

系统根据视频时长、帧率、预设模式和画面运动量动态计算抽帧目标。

抽帧参数包括：

```text
target_frame_count
min_frame_interval
max_frame_interval
sampling_fps
motion_threshold
blur_threshold
similarity_threshold
```

不同任务模式对应不同抽帧目标：

```text
fast：低帧数，快速预览
standard：中等帧数，稳定建模
high：高质量关键帧，质量优先
```

### 5.2 时间均匀采样

第一轮抽帧采用时间均匀采样，保证视频全程覆盖。

输出：

```text
raw_frames/
frames_manifest.json
```

`frames_manifest.json` 记录：

```text
frame_id
timestamp
source_frame_index
image_path
width
height
blur_score
motion_score
similarity_score
selected_status
drop_reason
```

### 5.3 运动自适应采样

系统根据画面运动量调整抽帧密度：

```text
运动慢的片段降低抽帧密度
运动快但画面稳定的片段提高抽帧密度
剧烈抖动片段降低权重
快速旋转片段降低权重
高重叠稳定片段保留关键帧
```

## 6. 坏帧清洗层

### 6.1 模糊帧过滤

通过图像清晰度算法识别模糊帧。

过滤类型包括：

```text
运动模糊
对焦失败
压缩噪声严重
画面拖影
低光噪点严重
```

### 6.2 重复帧过滤

通过相似度计算移除高度重复帧。

处理逻辑：

```text
计算相邻帧视觉相似度
保留视角变化有效的帧
移除几乎无变化的连续帧
避免大量重复帧进入 COLMAP
```

### 6.3 无效画面过滤

移除不适合参与建模的帧：

```text
大面积天空
大面积地面
大面积水面
大面积白墙
严重过曝
严重欠曝
强反光画面
遮挡画面
纯动态人群画面
快速转场画面
```

### 6.4 动态区域标记

识别视频中的动态区域，并生成动态掩码。

动态区域包括：

```text
行人
车辆
树叶
水面
烟雾
旗帜
强阴影变化
临时移动物体
```

输出：

```text
dynamic_masks/
dynamic_ratio.json
```

动态掩码用于后续相机注册、训练降权和质量诊断。

## 7. 关键帧选择层

### 7.1 关键帧评分

每一帧生成综合评分。

评分字段：

```text
sharpness_score
overlap_score
viewpoint_score
texture_score
exposure_score
dynamic_penalty
coverage_score
final_score
```

### 7.2 关键帧筛选

系统按照以下规则筛选关键帧：

```text
保证时间覆盖
保证空间视角变化
保证相邻帧重叠
保证清晰度
减少重复画面
减少动态干扰
保留闭环视角
```

### 7.3 关键帧分组

关键帧按时间和视角关系分组：

```text
primary_sequence：主连续序列
loop_candidates：闭环候选帧
side_view_candidates：侧向视角候选帧
weak_texture_segments：弱纹理片段
risk_segments：高风险片段
```

## 8. 相机位姿求解层

### 8.1 COLMAP 视频模式

视频输入使用顺序匹配策略。

核心配置：

```text
single_camera = true
camera_model = SIMPLE_RADIAL / OPENCV
matcher = sequential_matcher
guided_matching = true
loop_detection = true
```

流程：

```text
feature_extractor
→ sequential_matcher
→ mapper
→ image_undistorter
→ transforms.json 生成
```

### 8.2 COLMAP 参数增强

针对视频建模增强以下参数：

```text
提高 SIFT 特征数量
开启 guided matching
增加 sequential overlap
开启 loop detection
控制最大图像尺寸
统一相机内参
启用畸变模型
保留匹配日志
```

### 8.3 注册结果解析

解析 COLMAP 和 Nerfstudio 输出结果。

关键指标：

```text
input_frame_count
registered_frame_count
registration_ratio
camera_count
sparse_point_count
mean_reprojection_error
track_length_mean
coverage_ratio
failed_frame_count
```

### 8.4 位姿质量门禁

相机位姿求解后执行门禁检查。

检查项：

```text
注册帧数量
注册帧比例
重投影误差
相机轨迹连续性
稀疏点数量
场景覆盖范围
是否存在轨迹断裂
是否存在大量孤立帧
```

门禁结果：

```text
PASS：进入训练
WARN：进入训练并记录风险
FAIL：停止训练并输出诊断
```

## 9. 位姿求解增强层

### 9.1 自动重采样

当注册结果不达标时，系统自动触发重采样流程。

重采样策略：

```text
降低抽帧数量
提高关键帧质量阈值
增加帧间间隔
强化闭环候选帧
移除弱纹理片段
移除高动态片段
调整 sequential overlap
```

### 9.2 多策略注册

系统保留多套 COLMAP 注册策略。

策略类型：

```text
video_sequential
video_sequential_loop
video_exhaustive_small
video_vocab_tree
high_quality_keyframes
low_motion_keyframes
```

每次注册产出独立目录：

```text
colmap_attempt_001/
colmap_attempt_002/
colmap_attempt_003/
```

系统根据注册质量选择最优结果进入训练。

### 9.3 学习式位姿补强

当 COLMAP 注册失败或覆盖不足时，进入学习式位姿补强流程。

可接入模块：

```text
DUSt3R
MASt3R
VGGSfM
RoMa
LoFTR
SuperGlue
```

补强流程：

```text
COLMAP 初次注册
→ 注册质量检查
→ 学习式匹配 / 位姿估计
→ 生成候补相机位姿
→ 对齐相机坐标系
→ 质量检查
→ 输出 transforms.json
```

## 19. 分阶段落地链路

### Phase 1：基础自动建模链路

```text

COLMAP sequential 注册

```

### Phase 2：视频建模成功率补强

