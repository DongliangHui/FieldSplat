# 项目业务能力入场调研问卷

适用项目：公安应急处突指挥车系统、三维可视化移动指挥平台、刑事案件现场生成式实景 AI 三维重建及应用系统  
用途：帮助半路加入项目的人，在已经阅读项目书和宣传手册后，快速问清楚“产品现在到底具备哪些业务能力、哪些只是规划、哪些可以验收”。  
范围：只问业务能力、业务流程、使用边界、验收口径和当前进展；不问代码、技术栈、部署、算法实现细节。

## 1. 调研目标

本问卷围绕项目材料中的核心承诺展开：

1. 指挥车到达现场后，是否能快速掌握现场情况。
2. 是否能解决公安应急处突中的六类痛点：现场掌握滞后、异常行为难识别、重点人员识别困难、预警机制不足、情报更新不及时、处置方案不精准。
3. 是否能支撑群体性事件、大型活动安保、公共安全事件、刑事案件现场重建和灾害应急。
4. 是否能把无人机、车载视频、亿级成像、三维重建、行为识别、通信保障和指挥调度合成一个业务闭环。
5. 是否能把处置过程、重点人员轨迹、原始视频和三维场景沉淀为可复盘、可举证的材料。

调研时不要问“用了什么技术实现”，只问：

```text
能不能用？
谁来用？
什么场景用？
用到什么程度？
有什么结果输出？
能不能复核？
能不能验收？
现在做到哪一步？
```

## 2. 业务能力状态标尺

所有能力建议按以下状态填写，避免“基本可以”“差不多”等模糊说法。

| 状态 | 含义 |
| --- | --- |
| 未定义 | 只在材料中出现，没人说清楚业务流程和使用方式 |
| 已定义 | 已说清楚业务目标、用户、流程和验收口径 |
| 可演示 | 可以用样例数据或演示环境展示给外部看 |
| 可试用 | 真实用户能按流程试用，但还有缺口 |
| 可验收 | 有明确指标、真实场景或演练证明，可以进入验收 |
| 已交付 | 已在实际单位或真实任务中使用并形成记录 |

每项能力都要追问：

1. 当前状态是哪一级？
2. 谁确认的？
3. 有什么业务证据？
4. 离下一状态差什么？

## 3. 项目口径与最终产品形态

这些问题用于先统一“我们到底在做什么”。

1. 现在对外正式产品名称是什么？
2. 对客户介绍时，第一句话怎么说这个产品？
3. 当前产品最终形态是“公安应急处突指挥车系统”“三维可视化移动指挥平台”“刑事案件现场三维重建系统”，还是三者合一？
4. 如果三者合一，主线场景到底是哪一个？
5. 当前领导、客户、销售、产品、研发对产品定位是否一致？
6. 宣传手册中提到的 TH-GA04/XF-YJ04 是否就是当前要交付的产品型号？
7. “给硬件配上思考的大脑”这句话在业务上具体指哪些能力？
8. 第一阶段要让指挥车从“能看视频”升级到什么程度？
9. 当前最重要的业务成果是现场指挥、重点人员发现、三维重建、刑侦举证，还是综合演示？
10. 哪些能力是必须交付，哪些只是后续愿景？

## 5. 公安应急处突六类痛点

宣传材料中明确列出公安应急处突的痛点。逐项问清楚当前产品如何解决。

### 5.1 现场情况掌握滞后

1. 指挥车到达现场后，多久能让指挥员看到现场整体态势？
2. 当前能看到哪些信息：无人机视频、车载视频、地图、三维场景、设备状态、人群分布？
3. 指挥员是否能在一个界面看到全局？
4. 是否能快速定位重点区域？
5. 是否能查看多个方向、多个高度、多个视角？
6. 当前是否能覆盖建筑物遮挡、人群遮挡、复杂地形等问题？
7. 如果现场有盲区，系统如何提示？
8. 业务上如何判断“现场情况已被掌握”？
9. 这个能力当前是可演示、可试用还是可验收？

### 5.2 异常行为难以识别

1. 当前业务上需要识别哪些异常行为？
2. 项目材料中的奔跑、聚集、打斗、挥舞器械、非法拍摄、推搡、跌倒，哪些已纳入第一阶段？
3. 系统发现异常行为后，给谁提示？
4. 提示内容是什么：文字、画面框选、视频片段、风险等级、处置建议？
5. 是否有人复核异常行为？
6. 复核后结果如何记录？
7. 系统是否能区分一般聚集和高风险聚集？
8. 误报时业务流程怎么处理？
9. 漏报时如何追责或改进？
10. 当前有没有真实或模拟视频证明这些行为识别能力？

### 5.3 重点人员识别困难

1. 当前业务上“重点人员”指哪些人：挑头闹事者、在逃人员、前科人员、煽动者、行凶者、打砸抢者，还是授权名单人员？
2. 第一阶段是否真的要做人脸候选比对？
3. 如果做人脸候选，比对结果在业务上叫什么：候选、疑似、命中、确认？
4. 谁有权确认身份？
5. 是否要求人工复核？
6. 系统是否保留候选来源画面？
7. 候选结果是否能反查原始视频？
8. 低质量画面是否会提示“不适合识别”？
9. 如果无法做人脸比对，是否用轨迹、服饰、行为和位置来辅助锁定重点目标？
10. 当前是否已有重点人员发现的演示案例？

### 5.4 预警机制不完善

1. 当前系统要预警哪些风险？
2. 群体事件中，预警重点是聚集、拥堵、冲突、踩踏、通道阻断还是重点目标行为？
3. 大型活动中，预警重点是人流密度、入口拥堵、儿童走失、异常事件还是车辆聚集？
4. 灾害应急中，预警重点是烟火、积水、道路阻断、建筑损伤还是救援通道？
5. 每类预警是否有等级？
6. 每类预警是否有触发条件？
7. 预警后是否自动生成处置建议？
8. 谁确认预警有效？
9. 预警是否能关闭、升级、转任务？
10. 当前有没有预警闭环演示？

### 5.5 情报更新不及时

1. 现场信息多久刷新一次？
2. 指挥员能否看到最新视频、最新目标、最新风险和最新任务回执？
3. 情报员更新的信息如何同步给指挥员？
4. 后方指挥中心能否同步看到前方态势？
5. 现场发生变化后，系统如何提醒？
6. 重点人员移动后，轨迹是否持续更新？
7. 无人机发现新情况后，是否自动进入事件时间线？
8. 通信中断时，哪些情报留在本地，哪些恢复后补传？
9. 当前是否有“情报从发现到指挥员看到”的时间指标？

### 5.6 处置方案不够精准

1. 系统现在能给出哪些处置建议？
2. 建议是风险提示、区域管控建议、无人机侦察建议、广播建议、警力调度建议，还是报告建议？
3. 建议由系统自动生成，还是由情报员整理后提交？
4. 指挥员如何采纳或驳回建议？
5. 处置指令是否能直接下发给现场人员？
6. 是否支持分组广播？
7. 是否支持任务回执？
8. 处置方案和现场结果是否能在复盘中对应起来？
9. 当前有没有完整的“发现风险 -> 形成建议 -> 下达指令 -> 回执 -> 复盘”业务闭环？

## 20. 业务红旗问题

以下情况说明业务理解或项目状态存在重大风险。

1. 说不清第一阶段到底验收哪个场景。
2. 说不清项目是应急处突、刑侦重建还是综合指挥平台。
3. 所有人都说“能做”，但没人能说清楚业务流程。
4. 宣传手册里的能力没有对应演示或验收计划。
5. 重点人员识别没有授权、候选、复核和审计边界。
6. 风险预警没有等级、触发条件和处置动作。
7. 指挥调度只有展示，没有任务回执。
8. 三维重建只有模型展示，不能反查原始素材。
9. 复盘报告没有样例。
10. 证据链只停留在口头描述，没有事件级反查。
11. 后方中心同步没有明确业务对象。
12. 硬件已经准备，但软件业务流程还没有定稿。
13. 所有场景都想做，但没有第一优先级。
14. 验收指标无法用一次演练证明。
